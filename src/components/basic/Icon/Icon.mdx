#### Details

The `Icon` component provides a flexible and scalable way to integrate icons into your application. It leverages Material-UI's `SvgIcon` while extending support for custom sizes.

#### Interaction

Icons can be used standalone or as part of other components, such as buttons or lists. They serve as visual indicators and should be recognizable and consistent across the application.

#### Context and Usage

Use the `Icon` component to render MUI icons that are scalable and customizable. Icons can represent actions, objects, or navigation elements. Ensure icons are used consistently and are intuitive for the user.

#### Guidance

- **Size Variants**: The `Icon` component supports several size variants (`8`, `10`, `12`, `14`, `16`, `18`, `20`). Use these sizes according to the design specifications to maintain visual hierarchy.
- **Color Customization**: Icons can be colored using the `color` prop, which supports standard MUI color schemes ( `inherit`, `error`, `default`, `success`, `warning`). Ensure the color chosen aligns with the overall design theme.  
By default, if no color is specified, the icon will `inherit` the color from its parent.
- **Accessibility**: Always provide a descriptive `aria-label` for screen readers, especially when the icon conveys an important action or meaning.
- **Scalability**: The `Icon` component is designed to be easily extendable with additional icon libraries (currently configured with MUI icons only). Use the `iconName` prop to switch between different icon sets without altering the component's implementation.
- **Error Handling**: If an invalid `iconName` is provided, the component will log a warning and render nothing. This behavior ensures that the application fails gracefully without breaking the UI.

#### Example Usage

```jsx
<Icon
  iconName="Search"
  fontSize="16"
  color="success"
  aria-label="Search icon"
/>
```
